# AccessPilot MVP 30 天执行计划

本文将 `ACCESSPILOT_MVP_TASKS.md` 转成可执行的 4 周冲刺方案，确保在 30 天内达到“可收费上线”标准。

## 1. 里程碑（Milestones）

- **M1（第 1 周结束）**：完成项目基建、鉴权、组织、站点接入、手动触发扫描。
- **M2（第 2 周结束）**：完成 Playwright 抓取 + axe 扫描 + 问题入库 + 健康分。
- **M3（第 3 周结束）**：完成报告页、筛选、趋势、PDF 导出、LLM 基础解释。
- **M4（第 4 周结束）**：完成订阅支付、通知、营销页、稳定性加固、部署上线。

## 2. 每周任务分解

## Week 1（基础闭环）
**目标**：用户注册 -> 添加站点 -> 能触发一次扫描任务（哪怕先返回占位结果）

- 基础设施：`T001-T006`
- 账户组织：`T007-T010`
- 站点接入：`T011-T015`
- 扫描任务起步：`T017-T020`

**周验收输出**
1. 可登录后台并创建默认组织。
2. 可新增站点并在列表展示。
3. 可点击“立即扫描”，状态从 `queued` -> `running` -> `completed/failed`。

## Week 2（扫描能力闭环）
**目标**：真实抓取页面并输出结构化无障碍问题。

- 任务与重试：`T021`（可先实现简单固定重试）
- 抓取引擎：`T023-T030`（`T029` 可暂用低优先）
- 规则扫描：`T031-T038`

**周验收输出**
1. 单次扫描可覆盖 10-50 页（受站点结构影响）。
2. issues 可入库并按严重程度统计。
3. 每次扫描可生成健康分（0-100）。

## Week 3（可读报告闭环）
**目标**：用户可以“看懂并导出”扫描结果。

- 后台展示：`T046-T055`
- PDF：`T056-T060`
- AI 增强基础：`T039-T042`（`T043` 可延后）

**周验收输出**
1. 站点详情可看概览、问题列表、页面详情、历史趋势。
2. Pro 用户可导出完整版 PDF，Free 用户受限。
3. 问题详情可看到业务影响 + 修复建议（支持降级）。

## Week 4（商业化与上线）
**目标**：形成“可注册、可付费、可持续扫描与通知”的上线版本。

- 订阅支付：`T061-T066`
- 通知与定时：`T067-T071`
- 营销页：`T073-T075`
- 稳定与上线：`T079-T088`

**周验收输出**
1. Stripe 支付、回调、权限控制可用。
2. 首扫通知 + 严重问题提醒 + 周报可发送。
3. 完成生产部署并通过真实站点试扫验收。

## 3. 关键依赖顺序

1. `T004`（数据库）-> `T008/T011/T017/T061`（核心数据模型）。
2. `T018`（任务队列）-> `T019/T021/T070`（触发、重试、定时）。
3. `T023`（Playwright）-> `T031`（axe 扫描）-> `T032-T038`（结构化与评分）。
4. `T063/T064`（支付状态与权限）-> `T060`（导出限制）与站点/扫描限额策略。
5. `T079/T080`（鉴权与限流）是上线前硬门槛。

## 4. 风险与应对

- **扫描耗时过高**：优先落实 `T025` 页面上限 + `T027` 白名单抓取。
- **误报噪音影响体验**：优先做 `T036/T037` 指纹与去重。
- **LLM 成本不稳定**：优先 `T044/T045`（缓存 + 降级）。
- **上线前回归问题**：用 `T083` 覆盖核心主路径。

## 5. Definition of Done（MVP 级）

当以下 6 条全部满足，视为可发布：
1. 用户可注册、登录、添加站点。
2. 用户可手动触发扫描并看到状态变化。
3. 系统可扫描 10-50 页面并入库问题。
4. 报告页可看健康分、问题列表与详情。
5. 付费用户可导出 PDF 并收到邮件通知。
6. 系统可按周自动触发扫描并更新结果。
